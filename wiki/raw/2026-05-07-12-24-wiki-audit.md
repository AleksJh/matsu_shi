## Key Changes

### 🔴 CRITICAL DISCREPANCIES (требуют исправления в первую очередь)

#### `architecture/overview.md`
- **WRONG:** "consisting of **6 primary services**" с перечислением 7 пунктов, включая "Telegram Bot" как отдельный сервис.
- **TRUTH:** Бот интегрирован в Backend-сервис. В `app/main.py` (lines 13-30) бот запускается как `asyncio.create_task(dp.start_polling(bot))` внутри FastAPI lifespan. Отдельного Docker-контейнера для бота нет. Итого: **5 сервисов** (backend, frontend, admin, postgres, redis, nginx = 6 контейнеров, но бот не является отдельным).

#### `infrastructure/docker.md` + `procedures/deployment.md`
- **WRONG:** Вики инструктирует "Update nginx.prod.conf with the correct domain", подразумевая что это легко. 
- **TRUTH:** В `docker/nginx/nginx.prod.conf` строки 28-31 содержат **захардкоженный домен `matsushi.xyz`**:
  ```
  ssl_certificate     /etc/letsencrypt/live/matsushi.xyz/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/matsushi.xyz/privkey.pem;
  ```
  Файл необходимо редактировать вручную перед каждым деплоем на новый домен. Это **критично для воспроизводимости деплоя**.

#### `backend/rag-pipeline.md`
- **WRONG:** "Top candidates are reranked using Jina Reranker v3"
- **TRUTH:** Jina получает **все** объединённые кандидаты (до 40), а не только "top". Ключевое слово: merge → dedup → rerank ALL → take top-10.

- **WRONG:** Логика роутинга показана как `score >= 0.65 → LLM_LITE_MODEL`
- **TRUTH:** Код (`app/agent/router.py:38`):
  ```python
  if max_retrieval_score < RETRIEVAL_SCORE_THRESHOLD or query_class == "complex":
      return LLM_ADVANCED_MODEL
  return LLM_LITE_MODEL
  ```
  Условие `query_class == "complex"` **полностью пропущено** в вики. Агент-роутер может выбрать ADVANCED модель даже при высоком score, если классификатор пометил запрос как сложный.

#### `security/overview.md`
- **WRONG:** JWT описан как "short-lived token"
- **TRUTH:** `config.py`: `JWT_EXPIRE_MINUTES: int = 1440` = **24 часа**. Это стандартный, но не short-lived срок. Формулировка вводит в заблуждение с точки зрения security-ревью.

### ✅ CONFIRMED ACCURATE (подтверждено, менять не нужно)

- Все пороги RAG: `RETRIEVAL_NO_ANSWER_THRESHOLD=0.30`, `RETRIEVAL_SCORE_THRESHOLD=0.65`
- Hybrid search: dense (pgvector cosine) + sparse (PostgreSQL FTS Russian `tsvector`)
- Jina Reranker v3, top-40 candidates → top-10 final
- 5-шаговая FSM регистрации механика в боте (FIO → Country → City → Email → Phone)
- Callback actions: approve/deny/ban + уведомление механика с кнопкой "Открыть Matsu Shi"
- HMAC-валидация Telegram WebApp (`aiogram.utils.web_app.safe_parse_webapp_init_data`)
- JWT: `role=mechanic`, `sub=telegram_user_id`; admin — bcrypt-пароль
- Rate limiting: 15 req/min per user
- Модели: `gemini-2.5-flash-lite` (lite), `gemini-3-flash-preview` (advanced), `qwen3-embedding-4b` (embed, dim=1024)
- Docker-стек: `pgvector/pgvector:pg16`, `redis:7-alpine`, `nginx:alpine`, `python:3.13-slim`
- Multi-stage Dockerfile с `uv`, dev/prod targets
- SSE streaming с `proxy_buffering off` в nginx
- Langfuse tracing, Cloudflare R2 для visual assets (WebP, 150 DPI)
- Ingestion pipeline: 6 шагов (Parse→Visual→Chunk→Enrich→Embed→Write), checkpoint flags (`--stop-after`, `--start-from`, `--save-artifacts`, `--rebuild-index`)
- Все основные поля в схемах users, admin_users, queries, diagnostic_sessions (частично)
- `AsyncSessionLocal` с `expire_on_commit=False`
- HNSW index с cosine distance для pgvector
- CASCADE delete для chunks при удалении document/user
